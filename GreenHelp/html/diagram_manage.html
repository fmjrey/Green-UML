Diagram Management

Creating a New Diagram
New diagrams can be created in primarily two ways. The first is through Eclipse's File->New... menu. This option will create an empty diagram in the directory with a name specified. The second is through an element's context menu in the Package Explorer View. This option will create a diagram, and also add the selected element(s) to the new diagram.

Adding Contents to Diagrams

New Types
New Java types can be created in a diagram using the palette tools, Green will generate corresponding Java code in the project.

New Relationships
To add relationships to types in the diagram, select one of the relationship tools. Then, select the source and target types to create a relationship between them. Green will automatically generate the necessary code to create the selected relationship.

New Elements
New elements of types (field or method) can be added through the 'Add Field' / 'Add Method' context menus.

Notes
A note can be drawn directly in the editor, and its contents can be changed by double-clicking on it. Notes are saved along with the rest of the contents of the editor. They are selectable and resizable. Their contents are stored only in the diagram file and are not referenced anywhere in the Java code.

Adding Existing Elements
Exsiting elements can be added to diagrams through the context menus of the specific element in Package Explorer.

Refactor
Elements in the diagram can invoke JDT's refactoring commands 'Move,' 'Rename,' and 'Change Method Signature.' These commands are found in the 'Refactor' submenu of the context menus.

Removing Types
Types can be removed from a diagram using the 'Backspace' key or the appropriate context menu command. This does not change anything in the file system.

Deleting Elements
Selectable element (types, methods, fields, relationships, etc.) in a Green diagram can be deleted permanently by using the 'Delete' key. Non-editable Java elements, however, cannot be deleted (e.g. if java.lang.String is in a diagram, it cannot be deleted).
Warning: Deleted elements CANNOT be restored through 'Undo' (with the exception of Notes, which can be restored).

Refresh
To manually refresh the contents of the editor, press the "f" key, or use 'Refresh Editor' in the context menu.

Saving Diagrams
Green diagrams are saved to the directory within the project specified when they are created. However, a 'Save As...' dialog can be brought up to save a diagram elsewhere or as a different format. Green supports several formats for saving its diagrams. The default format is a Green File (extension .grn). Diagrams can also be saved as images using either JPEG (.jpg) or GIF (.gif) formats. To access these formats, change the file type specified in the Save As dialog (how this is done varies across operating systems). 'Save As...' can be accessed through Eclipse's 'File' menu or the diagram's context menu.

Printing Diagrams
Green supports printing of its diagram. Print can be accessed through Eclipse's File menu or the diagram's context menu.
Note: Eclipse 3.4, and therefore Green, does not fully support printing in Linux.
